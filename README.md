# Contact Extractor

Сканер контактов со скриншотов (карточки 2GIS, визитки, контакты и т.п.):
перетащил файлы — получил таблицу с именем/организацией, телефоном, email и
адресом, без лишнего текста. Дубликаты (в том числе один и тот же контакт на
разных скриншотах) схлопываются в одну строку.

## Стек

- Бэкенд: FastAPI (Python) + Pillow (ресайз изображений перед отправкой в ИИ)
- Фронт: один HTML-файл с JS (без фреймворков), экспорт в XLSX через SheetJS (CDN)
- LLM: OpenRouter, модель `google/gemini-3.1-flash-lite` — самая дешёвая модель
  с vision и структурированным JSON-выводом, которой достаточно для чтения
  скриншотов (~$0.25 / $1.50 за 1М input/output токенов)

## Как это работает

1. Перетаскиваешь скриншоты (или целую папку) в зону дропа, либо выбираешь
   файлы кликом. Можно скидывать сколько угодно — фронт сам режет на батчи
   по 100 файлов и шлёт их на бэкенд последовательно, с прогресс-баром.
2. Бэкенд на каждый батч:
   - считает sha256 каждого файла — если несколько файлов в батче побайтово
     идентичны, ИИ вызывается только один раз, остальные берут готовый результат;
   - уменьшает картинку (до 1400px по длинной стороне) и кодирует в JPEG,
     чтобы не жечь токены и память;
   - параллельно (до 8 одновременных запросов) шлёт в Gemini 3.1 Flash Lite
     через OpenRouter с промптом, который просит извлечь **только**
     имя/организацию, телефон, email и адрес — всё остальное (часы работы,
     сайт, соцсети, отзывы, вакансии и т.п.) модель игнорирует.
3. После того как все батчи обработаны, фронт дедуплицирует результаты:
   номера телефона нормализуются (убираются +7/8/скобки/пробелы, остаются
   последние 10 цифр), и записи с совпадающим телефоном ИЛИ email
   объединяются в одну — для каждого поля берётся самое полное значение
   среди дублей.
4. Таблица с пагинацией (по 50 строк на страницу), кнопки «Скачать CSV» и
   «Скачать XLSX» экспортируют уже дедуплицированный результат.

API-ключ OpenRouter вводится прямо на странице и нигде не сохраняется —
сервер получает его только в момент запроса к модели.

## Установка

Python 3.9+

```bash
pip install -r requirements.txt
```

## Запуск локально

```bash
uvicorn main:app --reload
```

Открыть в браузере: http://127.0.0.1:8000

## Ограничения

- Максимум 100 файлов в одном батче (фронт сам разбивает большие партии).
- Поддерживаются форматы png/jpg/jpeg/webp/gif/bmp. HEIC (стандартный формат
  скриншотов на iPhone до экспорта) не читается — такие файлы нужно сначала
  сконвертировать в jpg/png.
- Каждый батч — отдельный HTTP-запрос к Render; при большом количестве
  батчей общее время сканирования линейно растёт (можно оценить: на 4000
  файлов и батч 100 — это 40 последовательных запросов).

## Деплой на Render (бесплатно, без Docker)

1. Залить проект в GitHub (см. раздел "Git" ниже).
2. На dashboard.render.com: **New +** → **Web Service**.
3. Подключить репозиторий `contact-extractor`.
4. Настройки (если Render не подхватит `render.yaml` автоматически):
   - Environment: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Create Web Service → Render соберёт и задеплоит, выдаст адрес вида
   `https://contact-extractor-xxxx.onrender.com`.

Файл `render.yaml` в корне проекта уже описывает эту конфигурацию.

### Важные особенности бесплатного тарифа Render

- Сервис "засыпает" без трафика — первый запрос после простоя грузится
  дольше (холодный старт).
- Оперативная память ограничена (512 МБ), поэтому батч сознательно
  ограничен 100 файлами — это держит нагрузку в безопасных пределах даже
  на бесплатном тарифе.
- Файлы нигде не сохраняются между запросами (ни на диске, ни в базе) —
  каждый батч обрабатывается и отдаётся сразу, поэтому передеплой или сон
  сервиса ничего не портят: просто прогоняешь сканирование заново.

## Git: как залить проект и обновлять его (автокоммит → автодеплой)

Первая заливка (если папка ещё не git-репозиторий):

```bash
cd ~/Desktop/contact-extractor
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/drk-supersonic/contact-extractor
git push -u origin main
```

Дальнейшие обновления (после правок в коде):

```bash
git add . && git commit -m "апдейт" && git push
```

У Render уже настроен автодеплой при пуше в основную ветку — как
только Render подключён к репозиторию (шаг 2-3 выше), каждый `git push`
в `main` автоматически пересобирает и обновляет сервис на render.com,
ничего дополнительно нажимать не нужно.
