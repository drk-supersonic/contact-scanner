# Interview Bot

**Живая версия: [https://interview-bot-cgjk.onrender.com/](https://interview-bot-cgjk.onrender.com/)**

Тестовое задание: страница с одним вопросом, ответ пользователя уходит в LLM,
которая либо задаёт один уточняющий вопрос, либо благодарит и завершает диалог.

## Стек

- Бэкенд: FastAPI (Python)
- Фронт: один HTML-файл с JS (без фреймворков)
- LLM: OpenRouter, модель `openai/gpt-5-mini`

## Установка

Python 3.9+

```bash
pip install -r requirements.txt
```

## Настройка

API-ключ OpenRouter вводится прямо на странице (поле над вопросом), отдельно
настраивать сервер не нужно — можно сразу переходить к запуску. Ключ передаётся
на бэкенд только в момент запроса к модели и нигде не сохраняется.

Получить ключ бесплатно: https://openrouter.ai/keys

## Запуск

```bash
uvicorn main:app --reload
```

Открыть в браузере: http://127.0.0.1:8000

## Структура

## Как это работает

1. Фронт при загрузке запрашивает `/api/question` и показывает вопрос.
2. Пользователь вводит свой OpenRouter API Key и пишет ответ.
3. По клику «Отправить» (или Enter без Shift) фронт шлёт `POST /api/respond`
   с телом `{"answer": "...", "api_key": "sk-or-...", "round": 0}`.
4. Бэкенд выбирает один из двух системных промптов в зависимости от `round`:
   - `round = 0` → `SYSTEM_PROMPT_FIRST`: модель сама решает, задать ли
     ровно один уточняющий вопрос (если ответ поверхностный) или
     поблагодарить и завершить (если ответ подробный);
   - `round >= 1` → `SYSTEM_PROMPT_FINAL`: лимит уточнений исчерпан,
     модели запрещено задавать ещё один вопрос при любых обстоятельствах,
     она обязана завершить разговор благодарностью.
5. Реакция модели подставляется в тот же блок вместо исходного вопроса:
   - если это уточняющий вопрос — блок помечается как «Уточнение:», поле
     ответа очищается, счётчик `round` увеличивается, пользователь отвечает
     ещё раз (запрос уйдёт уже с `round = 1`, что автоматически включит
     `SYSTEM_PROMPT_FINAL` на бэкенде);
   - иначе — блок помечается как «Итог:», поле ответа блокируется, интервью
     считается завершённым.

Ограничение «не больше одного уточняющего вопроса» из задания обеспечено
дважды: и системным промптом на бэкенде (`SYSTEM_PROMPT_FINAL` не оставляет
модели выбора), и логикой на фронте (после первого уточнения интерфейс
в любом случае переходит в состояние «Итог»).

При сетевой ошибке к OpenRouter — до 2 повторных попыток с задержкой.

## Деплой на Render (бесплатно, без Docker)

1. Залить проект в GitHub (см. раздел "Git" ниже).
2. На dashboard.render.com: **New +** → **Web Service**.
3. Подключить репозиторий `interview-bot`.
4. Настройки (если Render не подхватит `render.yaml` автоматически):
   - Environment: `Python 3`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Create Web Service → Render соберёт и задеплоит, выдаст адрес вида
   `https://interview-bot-xxxx.onrender.com`.

Файл `render.yaml` в корне проекта уже описывает эту конфигурацию — если Render
предложит "использовать Blueprint", можно просто подтвердить, шаги 4 пройдут сами.

Важно: бесплатный тариф "засыпает" без трафика. Открой ссылку за минуту до
записи демо-видео, чтобы сервис успел проснуться.

## Git: как залить проект и обновлять его

Первая заливка (если папка ещё не git-репозиторий):

```bash
cd ~/Desktop/interview-bot
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/drk-supersonic/interview-bot
git push -u origin main
```

Дальнейшие обновления (после правок в коде):

```bash
git add .
git commit -m "апдейт"
git push
```

Можно одной строкой:

```bash
git add . && git commit -m "апдейт" && git push
```

**Про `git push -f`**: это принудительный push, он ЗАТИРАЕТ историю на GitHub
версией с твоего компьютера. Нужен только если ты сознательно переписал историю
локально (например, `git commit --amend`) и знаешь, что удалённые коммиты можно
потерять. Для обычных обновлений он не нужен — обычный `git push` сделает то же
самое, но безопасно (не даст затереть чужие изменения, если они появятся).
Использовать `-f` "по умолчанию" не стоит.
